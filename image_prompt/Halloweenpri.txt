### Overlay & Composition — **STRICT**

* **Fixed Z-order (no exceptions):** Background → Characters → Confetti → Frame → Stickers → Doodles → Title → Date.
* **UI overlay domain:** Stickers and Doodles are always UI overlays on top of the scene. They must be drawn **only above** hair/skin/clothes, never integrated into their shading, normals, or material response.
* **Independent lighting:** Stickers/Doodles must **not** react to scene lighting. No self-shadowing, no environment reflections, no subConfetti scattering; render them as lighting-independent 2D marks.

### Stickers (embroidered patch / die-cut look) — **OVERRIDE**

* **Appearance:** **Die-cut sticker / embroidered patch** look with a **white kiss-cut border 1.5–3 px** and a **tiny drop shadow (1–2 px at 30–45% opacity)**.
* **Texture:** **Satin stitch / chain stitch** with **uniform, isotropic thread grain**. **Do not** synchronize with character shading. **No** localized metallic highlights.
* **Blend mode & opacity:** **Normal** blend only; **opacity fixed at 90–100%**. **Do not** use Multiply/Screen/Overlay, etc.
* **Non-interference:** **Do not** alter the silhouettes of hair/skin/garments. **No** 3D “wrap-around” across hair strands or forms.
* **Face safety:** **Do not** cover more than **15%** of any face’s visible area. **Never** obstruct pupils/irises, mouth, or nasal bridge.

### Doodles (chalk scribbles) — **OVERRIDE**

* **Appearance:** **Blackboard chalk / wax crayon** feel. Include natural **pressure variation** and **powdery chips**, but **no thick paint brushes**.
* **Line spec:** **Broken lines 1–3 px**, **weak anti-aliasing**, a touch of **Confetti jitter** like on paper/glass.
* **Blend & lighting:** **Normal** blend at **80–90% opacity**. **No** glow or rim-light; **do not** sync with character rim-lights.
* **Placement rule:** Follow **Randomizer #6**. However, **never draw on the character who is designated as “not receiving doodles.”** Doodles may cross silhouettes but **must not conform or wrap** to the 3D forms (no “wrap-around”).
* **Hard bans:** **No** tattoo-like skin assimilation, **no** makeup substitution, **no** turning doodles into physical attachments.

## Randomization Protocol (STRICT)
- Always perform randomization via Python in Code Interpreter (never simulate).
- If Python is unavailable, STOP and print: "ランダム化がブロックされました: Python が利用できません" (do not fabricate picks).

For every Randomizer list below:
1) Initialize a single run-wide RNG with an integer SEED (if provided; otherwise generate one and print it). Do NOT re-seed per list.
2) Shuffle each list uniformly (Fisher–Yates semantics; every permutation equally likely).
3) Select items without replacement, then apply the picks as binding instructions (not preferences).

### Selection rules
- Background, Pose, Confetti, Frame, Colorway: take the first item after shuffle.
- Stickers: let K ~ Uniform{3..6}; take the first K items (no duplicates). Record `stickers_count = K`.
- Doodle target (#6): take the first item.
- Doodles (#7): 
  - if target=None → take 0 items,
  - if target=CharX only → take 1 item,
  - if target=Both → take 2 items;
  items must be distinct. Record `doodles_count`.
- Text styles (#8): choose one for `text_title_style` and one for `text_date_style` independently (they may coincide).

### [Randomization Output]
Before any image generation, print a JSON object named `random_picks` containing:
`background, pose, Confetti, stickers[], stickers_count, doodle_target, doodles[], doodles_count, frame, text_title_style, text_date_style, colorway, seed`.

If any list has fewer available items than requested, cap the count and include `"note": "capped due to list length"`.

**Notes**
- The `seed` controls list shuffling only; it does NOT control the image model’s internal seed.
- Selections must be uniformly random, not preference-based.
